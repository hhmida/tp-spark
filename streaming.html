    Spark Streaming¶

    Spark offre la possibilité de traiter des données en flux à partir de plusieurs sources (kafka, twitter, TCP, ...). Spark transforme le flux continue de données en batchs à des intervalles de temps réguliers. Il crée ainsi une séquence continue de RDDs de même type accessibles à travers la classe DStream. Cette classe crée un RDD à des intervalles de temps réguliers.

    Le point d'entrée aux fonctionnalités de Streaming de Spark est le StreamingContext. Il permet de créer des instances de DStream.

    StreamingContext¶

    Création¶

    StreamingContext(sparkContext, batchDuration=None)
    Exemple
    from pyspark.sql import SparkSession
from pyspark.streaming import StreamingContext
spark = SparkSession.builder.appName('Exemple Streaming').getOrCreate()
ssc = StreamingContext(spark.sparkContext, 10)

    Principales Méthodes¶

    • socketTextStream(hostname, port) : flux de données à partir de socket TCP (codage UTF8 et \n pour fin de ligne).
    • textFileStream(directory) : surveille un dossier et lit les nouveaux fichiers dans le format texte.
    • binaryRecordsStream(directory, recordLength) : comme textFileStream mais les fichiers sont lus dans le format binaire avec des enregistrements de taille fixe = recordLength.
    • queueStream(rdds, oneAtATime=True, default=None) : une file de RDDs comme source de données.
    • start() : démarre le traitement des flux.
    • stop(stopSparkContext=True, stopGraceFully=False) : arrêt du traitement des flux.
    • awaitTermination(timeout=None) : attente de la fin d'exécution.

    DStream (Discretized Stream)¶

    Une instance DStream est caractérisée par :

    • La liste d'autres DStreams sur lesquelles elle dépend.
    • Un intervalle au bout duquel elle génère un RDD.
    • Une fonction utilisée pour la génération du RDD.

    DStream possède plusieurs méthodes communes avec RDD comme : `map(), flatMap(), reduce(), reduceByKey(), filter(), count(), join(), ... Elles générent également des DStreams.

    Autres méthodes :

    • pprint([num]) : affiche les n premiers éléments.
    • window(windowDuration[, slideDuration]) : crée un nouveau DStream où chaque RDD contient les éléments reçus dans une fenêtre de temps sur le DStream.

    Exemples¶

    Dans ce qui suit nous montrons comment traiter un flux continu de messages en provenance de Kafka. Install and Setup Kafka Cluster

    Spark Streaming avec socketTextStream¶

    Démarrer netcat

    nc -l -p 9999

    Démarrer l'application Spark

    spark-submit network_wordcount.py localhost 9999
    network_wordcount.py
    import sys

from pyspark import SparkContext
from pyspark.streaming import StreamingContext

if __name__ == "__main__":
    if len(sys.argv) != 3:
        print("Usage: network_wordcount.py <hostname> <port>", file=sys.stderr)
        sys.exit(-1)
    sc = SparkContext(appName="PythonStreamingNetworkWordCount")
    ssc = StreamingContext(sc, 5)

    lines = ssc.socketTextStream(sys.argv[1], int(sys.argv[2]))
    counts = lines.flatMap(lambda line: line.split(" "))\
                .map(lambda word: (word, 1))\
                .reduceByKey(lambda a, b: a+b)
    counts.pprint()

    ssc.start()
    ssc.awaitTermination()

    Spark Streaming avec Kafka¶

    Installation de Kafka¶

    Si kafka n'est pas installé alors suivre les étapes suivantes :

    Télécharger les binaires de Kafka

    wget https://archive.apache.org/dist/kafka/0.10.2.2/kafka_2.12-0.10.2.2.tgz

    Décompresser et renommer le dossier obtenu

    tar -xzf kafka_2.12-0.10.2.2.tgz
mv kafka_2.12-0.10.2.2.tgz kafka

    Ajouter l'emplacement de Kafka à la variable PATH dans .bashrc ou .profile

    echo "PATH=$PATH:~/kafka/bin" >> ~/.bashrc
source ~/.bashrc
    Démarrer l'exemple¶

    Les commandes sont exécutées à partir du dossier kafka

    Démarrer Zookeeper

    bin/zookeeper-server-start.sh config/zookeeper.properties
    Zookeeper

    ZooKeeper est un service de coordination pour les applications distribuées.ZooKeeper enregistrer les métadonnées concernant le cluster (hôtes, ports, ...). Il est fourni avec Kafka.

    Démarrer le broker Kafka dans une nouvelle session

    bin/kafka-server-start.sh config/server.properties

    Créer un topic sur Kafka Dans une nouvelle session :

    bin/kafka-topics.sh --create --zookeeper localhost:2181 \
--replication-factor 1 \
--partitions 1 \
--topic wordcounttopic
    Topic Kafka

    Les messages Kafka sont organisés en topics. Ces derniers sont partitionnés et répliquées sur de multiples brokers à travers le cluster. Un processus Producer envoie les messages au topic et un processus consumer les lit. Dans cet exemple, le producer est le programme fourni par Kafka, le consumer est Spark. Pour voir les topics disponibles :

    bin/kafka-topics.sh --zookeeper localhost:2181 --list

    Démarrer le producer dans une nouvelle session

    bin/kafka-console-producer.sh --broker-list localhost:9092 --topic wordcounttopic

    Démarrer l'application Spark

    spark-submit --master yarn --deploy-mode client --conf "spark.dynamicAllocation.enabled=false" \
--packages org.apache.spark:spark-streaming-kafka-0-8_2.11:2.4.0 kafka_wordcount.py \
localhost:2181 wordcounttopic
    kafka_wordcount.py
    from __future__ import print_function
import sys
from pyspark import SparkContext
from pyspark.streaming import StreamingContext
from pyspark.streaming.kafka import KafkaUtils
if __name__ == "__main__":
    if len(sys.argv) != 3:
        print("Usage: kafka_wordcount.py <zk> <topic>", file=sys.stderr)
        sys.exit(-1)
    sc = SparkContext(appName="PythonStreamingKafkaWordCount")
    ssc = StreamingContext(sc, 10)
    zkQuorum, topic = sys.argv[1:]
    kvs = KafkaUtils.createStream(ssc, zkQuorum, "spark-streamingconsumer", {topic: 1})
    lines = kvs.map(lambda x: x[1])
    counts = lines.flatMap(lambda line: line.split(" "))\
        .map(lambda word:(word, 1)).reduceByKey(lambda a, b: a+b)
    counts.pprint()
    ssc.start()
    ssc.awaitTermination()

    Structured Streaming¶

    Spark Structured Streaming offre un moteur de traitement de flux au-dessus de Spark SQL. Il permet d'exprimer les traitements sur les flux de la même façon que sur des données statiques. Spark SQL se charge de l'exécution continue et incrémentale et la mise à jour du résultat final. Spark réalise ce traitement avec une latence de l'ordre de 1 milliseconde. Les flux de données sont représentées sous forme de DataFrame qui peut être assimilée à une table de taille illimitée dont le contenu augmente au cours du temps avec la réception du flux.

    L'API Spark, repose sur la classe DataStreamReder pour charger un DataFrame de flux à partir d'une source cpmme HDFS, Kafka, Kinesis, ...). Le traitement sur ce flux est exprimé via une requête StreamingQuery. L'écriture d'un flux est prise en charge par la classe DataStreamWriter obtenu à partir de DataFrame.writeStream.

    Le modèle de programmation est représenté dans la figure suivante :

    L'instance DataStremReader est obtenu à partir de SparkSession par : readStream. Elle possède les méthodes :

    • format(source) : spécifie la source (socket, kafka, ...)
    • option(key, value) : ajoute une configuration pour la source
    • csv(path[, schema, sep, encoding, quote, …]) : charge un fichier csv
    • text(path[, wholetext, lineSep, …]) : charge un fichier texte
    • load([path, format, schema]) : charge le flux de la source et le retourne sous la forme de DataFrame

    Et voici quelques méthode la classe DataStreamWriter :

    • format(source) : spécifie la source (socket, kafka, ...)
    • option(key, value) : ajoute une configuration pour la source
    • start([path, format, outputMode, …]) : charge le flux de la source et le retourne sous la forme de DataFrame

    • outputMode(outputMode) : append, complete, ...

    • awaitTermination([timeout])

    • stop()
    • processAllAvailable()

    Exemple¶

    Suivre les mêmes étapes qua dans l'exemple socketTextStream.

    structured_network_wordcount.py
    import sys, os, json
#    Spark
from pyspark.sql import SparkSession
#    Spark Streaming
from pyspark.streaming import StreamingContext
from pyspark.sql.functions import explode
from pyspark.sql.functions import split

spark = SparkSession.builder.appName('Spark Structured Streaming').getOrCreate()
# Create SPARK Context
sc = spark.sparkContext
sc.setLogLevel("WARN")
## Create Streaming context , with 10 second interval 
ssc = StreamingContext(sc,  10) 
# Create DataFrame representing the stream of input lines from connection to localhost:9999
lines = spark \
    .readStream \
    .format("socket") \
    .option("host", "localhost") \
    .option("port", 9999) \
    .load()

# Split the lines into words
words = lines.select(
explode(
    split(lines.value, " ")
).alias("word")
)

# Generate running word count
wordCounts = words.groupBy("word").count()
# Start running the query that prints the running counts to the console
query = wordCounts \
    .writeStream \
    .outputMode("complete") \
    .format("console") \
    .start()
query.awaitTermination()
