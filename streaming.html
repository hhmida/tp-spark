    Spark Streaming¶

    Spark offre la possibilité de traiter des données en flux à partir de plusieurs sources (kafka, twitter, TCP, ...).

    Dans ce qui suit nous montrons comment traiter un flux continu de messages en provenance de Kafka. Install and Setup Kafka Cluster

    Spark Streaming avec socketTextStream¶

    Démarrer netcat

    nc -l -p 9999

    Démarrer l'application Spark

    spark-submit network_wordcount.py localhost 9999
    network_wordcount.py
    import sys

from pyspark import SparkContext
from pyspark.streaming import StreamingContext

if __name__ == "__main__":
    if len(sys.argv) != 3:
        print("Usage: network_wordcount.py <hostname> <port>", file=sys.stderr)
        sys.exit(-1)
    sc = SparkContext(appName="PythonStreamingNetworkWordCount")
    ssc = StreamingContext(sc, 5)

    lines = ssc.socketTextStream(sys.argv[1], int(sys.argv[2]))
    counts = lines.flatMap(lambda line: line.split(" "))\
                .map(lambda word: (word, 1))\
                .reduceByKey(lambda a, b: a+b)
    counts.pprint()

    ssc.start()
    ssc.awaitTermination()

    Spark Streaming avec Kafka¶

    Installation de Kafka¶

    Si kafka n'est pas installé alors suivre les étapes suivantes :

    Télécharger les binaires de Kafka

    wget http://apache.claz.org/kafka/2.1.0/kafka_2.11-2.1.0.tgz

    Décompresser et renommer le dossier obtenu

    tar -xzf kafka_2.11-2.1.0.tgz
mv kafka_2.11-2.1.0.tgz kafka

    Ajouter l'emplacement de Kafka à la variable PATH dans .bashrc ou .profile

    echo "PATH=$PATH:~/kafka/bin" >> ~/.bashrc
source ~/.bashrc

    Démarrer l'exemple¶

    Les commandes sont exécutées à partir du dossier kafka

    Démarrer Zookeeper

    bin/zookeeper-server-start.sh config/zookeeper.properties
    Zookeeper

    ZooKeeper est un service de coordination pour les applications distribuées.ZooKeeper enregistrer les métadonnées concernant le cluster (hôtes, ports, ...). Il est fourni avec Kafka.

    Démarrer le broker Kafka dans une nouvelle session

    bin/kafka-server-start.sh config/server.properties

    Créer un topic sur Kafka Dans une nouvelle session :

    bin/kafka-topics.sh --create --zookeeper localhost:2181 \
--replication-factor 1 \
--partitions 1 \
--topic wordcounttopic
    Topic Kafka

    Les messages Kafka sont organisés en topics. Ces derniers sont partitionnés et répliquées sur de multiples brokers à travers le cluster. Un processus Producer envoie les messages au topic et un processus consumer les lit. Dans cet exemple, le producer est le programme fourni par Kafka, le consumer est Spark. Pour voir les topics disponibles :

    bin/kafka-topics.sh --zookeeper localhost:2181 --list

    Démarrer le producer dans une nouvelle session

    bin/kafka-console-producer.sh --broker-list localhost:9092 --topic wordcounttopic

    Démarrer l'application Spark

    spark-submit --master yarn --deploy-mode client --conf "spark.dynamicAllocation.enabled=false" \
--packages org.apache.spark:spark-streaming-kafka-0-8_2.11:2.4.0 kafka_wordcount.py \
localhost:2181 wordcounttopic
    kafka_wordcount.py
    from __future__ import print_function
import sys
from pyspark import SparkContext
from pyspark.streaming import StreamingContext
from pyspark.streaming.kafka import KafkaUtils
if __name__ == "__main__":
    if len(sys.argv) != 3:
        print("Usage: kafka_wordcount.py <zk> <topic>", file=sys.stderr)
        sys.exit(-1)
    sc = SparkContext(appName="PythonStreamingKafkaWordCount")
    ssc = StreamingContext(sc, 10)
    zkQuorum, topic = sys.argv[1:]
    kvs = KafkaUtils.createStream(ssc, zkQuorum, "spark-streamingconsumer", {topic: 1})
    lines = kvs.map(lambda x: x[1])
    counts = lines.flatMap(lambda line: line.split(" "))\
        .map(lambda word:(word, 1)).reduceByKey(lambda a, b: a+b)
    counts.pprint()
    ssc.start()
    ssc.awaitTermination()

    Messages de log

    Si Spark affiche trop de messages d'information, vous pouvez réduire le niveau de journalisation en modifiant la ligne du fichier spark/conf/log4j.properties :

    log4j.rootCategory = ERROR, console
