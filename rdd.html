    Manipuler les RDD¶

    Les RDD prennent en charge deux types d'opérations: les transformations, qui créent un nouvel RDD à partir d'un RDD existant, et les actions, qui renvoient une valeur au programme Driver après avoir exécuté un calcul sur RDD.

    Toutes les transformations dans Spark sont paresseuses, en ce sens qu'elles ne calculent pas leurs résultats tout de suite. Au lieu de cela, ils se souviennent simplement des transformations appliquées à un ensemble de données de base (par exemple, un fichier). Les transformations ne sont calculées que lorsqu'une action nécessite qu'un résultat soit renvoyé au programme pilote.

    Par défaut, chaque RDD transformé peut être recalculé chaque fois que vous exécutez une action dessus. Cependant, vous pouvez également conserver un RDD en mémoire à l'aide de la méthode persist (ou cache), auquel cas Spark conservera les éléments sur le cluster pour un accès beaucoup plus rapide la prochaine fois que vous l'interrogerez. Il existe également une prise en charge des RDD persistants sur disque ou répliqués sur plusieurs nœuds.

    Transformations¶

    TransformationDescription
    map(func) Retourne un RDD avec les éléments obtenus de l'application de func sur les éléments deu RDD intial.
    filter(func) Construit un RDD avec les éléments sur lesquels func retourne True.
    flatMap(func) Comme map mais la fonction func retourne une liste d'éléments associée à chaque élément du RDD initial.
    mapPartitions(func) Similar to map, but runs separately on each partition (block) of the RDD, so func must be of type Iterator<T> => Iterator<U> when running on an RDD of type T.
    mapPartitionsWithIndex(func) Comme mapPartition et fournit le numéro de la partition à utiliser.
    sample(withReplacement, fraction, seed) Sélectionner une fraction des données, avec ou sans remise.
    union(otherDataset), intersection Opérations ensemblistes.
    distinct([numPartitions])) Retourne un RDD contenant les valeurs uniques.
    reduceByKey(func, [numPartitions]) Appelée sur un RDD de paires (K, V) et retourne aussi un RDD de apires (K, V) où les valeurs sont agrégées selon la fonction func. Le nombre de tâches reduce est déterminée par le second paramètre.
    aggregateByKey(zeroValue)(seqOp, combOp, [numPartitions]) Permet d'obtenir un résultat d'agrégation de type différent de celui des valeurs agrégées.
    sortByKey([ascending], [numPartitions]) Tri selon la clé.
    join(otherDataset, [numPartitions]) Jointure de deux RDDs de paires (K, V) and (K, W) qui retourne une RDD de paires (K, (V, W)). La jointure externe est réalisée avec leftOuterJoin, rightOuterJoin, et fullOuterJoin.
    cartesian(otherDataset) Produit cartésien de deux RDDs.
    coalesce(numPartitions) Diminuer le nombre de partitions. Utile suite à un filtrage du RDD.

    Actions¶

    ActionDescription
    reduce(func) Applique la fonction d'agrégation func.
    collect() Retourne les éléments du RDD sous la frorme de liste au Driver.
    count() Retourne le nombre d'éléments du RDD.
    first() Retourne le premier élément (équivalent à take(1)).
    take(n) Retourne un tableau des n premiers éléments.
    takeSample(withReplacement, num, [seed]) Combinaison de sample et take.
    takeOrdered(n, [ordering]) Return the first n elements of the RDD using either their natural order or a custom comparator.
    saveAsTextFile(path) Enregistre les éléments dans un fichier texte (chaque élément est converti en chaîne de caractères).
    countByKey() Retourne le nombre d'éléments pour chaque clé.
    foreach(func) Applique une fonction sur chaque élément sans retour. Par exemple la sauvegarde dans un fichier.

    Persistance des RDD¶

    C'est la clé pour le gain de performance lors des traitements itératifs sur les RDDs. Il s'agit de garder en mémoire les données pour de futures manipulations (voir l'exemple dans la section précédente).

    Pour activer la persistance sur un RDD, il suffit d'appleer persist() ou cache(). Il est possible de définir des niveaux de persistance avec persist(StorageLevel.<niveau>) ou niveau peut être :

    • MEMORY_ONLY : En mémoire. Si la mémoire est insuffisante les partitions non chargées en mémoire seront calculées à la volée au besoin. C'est le niveau par défaut (utilisé avec cache()).
    • MEMORY_AND_DISK Les partitions qui déapssent la capacité de la mémoire sont stockées sur disque.
    • MEMORY_ONLY_SER et MEMORY_AND_DISK_SER (Java and Scala) : idem. mais les données sont sérialisés. Avec Python, les RDD sont toujours sérialisées au format pickle.
    • DISK_ONLY : stockage sur disque seulement.
    • MEMORY_ONLY_2, MEMORY_AND_DISK_2, etc. : avec réplication sur 2 noeuds.
